# CFIRE 艺人专业技能管理技能

## 功能定位

本技能让智能体能够自主识别技能差距、学习并记录专业技能知识，形成可复用的专业能力库。每个专业技能独立存储，便于检索和应用到内容创作、粉丝互动、产品设计等场景中。

## 核心能力

- **技能差距识别**：分析当前任务需求与已有技能的匹配度，识别需要补充的专业知识
- **自主学习研究**：针对识别到的技能差距，通过互联网搜索、资料整理等方式自主学习
- **技能文档生成**：将学习到的专业知识整理成结构化的技能文档
- **技能存储管理**：按领域分类存储技能文档，支持检索和版本管理
- **技能应用调用**：在内容生成、粉丝互动等任务中检索并应用相关专业技能

## 触发场景

当智能体出现以下意图时使用本技能：

- 需要在特定领域（如音乐制作、心理学、互联网运营等）生成专业内容
- 发现当前任务超出已有知识范围，需要补充专业技能
- 用户要求智能体学习某项新技能或专业知识
- 需要检索已有技能文档作为内容创作的参考
- 需要更新或完善已有技能文档的内容

## 目录结构

```text
assets/
  expertise/           # 专业技能文件目录
    *.md               # 独立技能文件，按领域命名

reference/
  EXPERTISE_SCHEMA.md      # 技能文件结构规范（所有技能共用）
  LEARNING_PROTOCOL.md     # 自主学习与技能获取协议
  EXPERTISE_INDEX.md       # 技能索引与检索目录
```

## 文件加载策略

### 按需加载

技能文件根据任务需求选择性加载：

| 文件 | 何时读取 |
| --- | --- |
| `EXPERTISE_SCHEMA.md` | 创建或验证技能文件时 |
| `LEARNING_PROTOCOL.md` | 识别技能差距或学习新技能时 |
| `EXPERTISE_INDEX.md` | 检索已有技能时 |
| `assets/expertise/*.md` | 需要应用特定领域技能时 |

### 检索策略

| 场景 | 推荐方法 |
| --- | --- |
| 查找特定领域技能 | 通过 `EXPERTISE_INDEX.md` 检索领域标签 |
| 查找与主题相关的技能 | 关键词匹配技能文件标题和描述 |
| 查找技能文件位置 | 通过索引定位文件路径 |

## 工作模式

### 模式一：技能差距识别

当智能体执行任务时，检测是否需要专业技能支持：

1. 分析任务需求，提取领域关键词
2. 检查 `EXPERTISE_INDEX.md` 是否存在相关技能
3. 如果不存在或技能不足，触发模式二

### 模式二：自主学习与技能生成

针对识别到的技能差距进行学习：

1. **需求分析**：明确需要学习的技能领域和具体目标
2. **信息收集**：通过互联网搜索、资料整理收集专业知识
3. **内容整理**：按照 `EXPERTISE_SCHEMA.md` 规范整理技能文档
4. **确认提交**：生成技能草案，请求用户确认后保存

### 模式三：技能应用

在内容生成等任务中应用已有技能：

1. 根据任务主题检索相关技能文件
2. 加载技能内容作为上下文参考
3. 确保生成内容符合专业知识要求

### 模式四：技能更新与维护

定期审查和更新技能文档：

1. 检查技能文档的时效性和准确性
2. 补充新的专业知识和实践经验
3. 更新 `EXPERTISE_INDEX.md` 索引信息

## 决策规则

- 技能文档的创建和修改必须遵循 `EXPERTISE_SCHEMA.md` 规范
- 智能体可自主起草技能文档，但正式保存前必须经过用户确认（L2 级别）
- 技能文档应保持独立性，不同领域技能分开存储
- 技能内容应基于可靠来源，避免编造专业知识
- 技能文档应包含版本信息和更新记录，便于追溯

## 文件修改权限

### 分级规则（强制执行）

#### 🔴 完全禁止智能体修改（稳定层）

所有稳定层文件为系统配置，智能体无任何写入权限，仅可读取：

- `reference/EXPERTISE_SCHEMA.md`：技能文件结构规范
- `reference/LEARNING_PROTOCOL.md`：学习协议
- `SKILL.md`：技能本身定义

#### 🟡 仅允许人工修改（索引层）

技能索引文件，智能体可提议修改，但必须经过用户确认：

- `reference/EXPERTISE_INDEX.md`：技能索引与检索目录

#### 🟢 允许智能体自行起草，人工确认后保存（技能层）

专业技能文件，智能体可自主起草和更新，但正式保存前必须用户确认：

- `assets/expertise/*.md`：各领域专业技能文件

### 权限校验规则

1. 智能体尝试修改🔴级文件时必须拒绝操作
2. 智能体尝试修改🟡级文件时必须先向用户发起确认请求
3. 🟢级文件修改必须符合 `EXPERTISE_SCHEMA.md` 规范
4. 所有文件修改操作必须记录操作日志，包含修改原因、修改内容、时间戳

## 工具协同

- 需要发布涉及专业技能的内容时，先通过 `cfire-artist-profile` 的 `REVIEW.md` 检查
- 技能学习过程中如需搜索互联网信息，可调用外部搜索工具
- 技能文档可作为 `cfire-artist-daily` 内容策划的专业知识参考
- 技能学习记录可沉淀到 `cfire-artist-profile` 的 `MEMORY.md`

## 技能加载与消费协议

其他技能在执行内容生成任务前，应按照以下协议发现和加载相关专业技能：

### 消费流程

1. **领域提取**：从任务需求中提取领域关键词（如：音乐治疗、心理学、视频制作等）
2. **技能检索**：读取 `EXPERTISE_INDEX.md`，查找匹配的技能条目
3. **技能加载**：根据索引中的文件路径，加载对应的 `assets/expertise/*.md` 技能文件
4. **上下文注入**：将技能内容作为专业知识上下文注入到内容生成过程中
5. **质量校验**：确保生成内容符合技能文档中的专业知识要求

### 消费场景

| 消费技能 | 消费时机 | 消费方式 |
| --- | --- | --- |
| `cfire-artist-profile` | 内容生成前、粉丝回复前 | 检索并加载相关领域技能作为上下文 |
| `cfire-artist-daily` | 内容策划前、日记生成前 | 检索并加载相关领域技能作为专业知识参考 |
| `cfire-artist-post` | 发布前审查 | 验证内容是否符合相关专业技能要求 |

### 检索优先级

1. **领域精确匹配**：优先匹配完全相同的领域名称
2. **标签模糊匹配**：其次匹配标签列表中的关键词
3. **描述语义匹配**：最后匹配技能描述中的相关词汇
