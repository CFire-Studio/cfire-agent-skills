# EXPERTISE_SCHEMA

## 用途

定义每个专业技能文件必须遵循的标准结构，确保技能文档的一致性和可检索性。

## 命名规范

### 文件名

- 文件名为中文技能名称，后缀 `.md`
- 使用简洁、准确的领域名称，避免过长
- 示例：`音乐疗愈技能.md`、`短视频制作技能.md`、`心理学基础技能.md`

### 命名约束

- 文件名不得包含特殊字符（`/ \ : * ? " < > |`）
- 文件名长度不超过 50 个字符
- 同一领域技能如需区分版本，在名称后加版本号，如 `音乐疗愈技能_v2.md`

## 文件结构

每个技能文件必须包含以下两个部分：

### 1. 元数据头部（必需）

```markdown
---
name: "<技能名称>"
domain: "<技能领域>"
proficiency: "<熟练度级别>"
acquisition_date: "<获取日期，YYYY-MM-DD>"
source: "<知识来源，如：互联网搜索、专业书籍、用户输入等>"
version: "<版本号，如：1.0>"
update_date: "<最后更新日期，YYYY-MM-DD>"
tags: ["<标签1>", "<标签2>", "<标签3>"]
description: "<一句话描述该技能的核心内容和用途>"
---
```

#### 字段说明

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 技能名称，应与文件名一致 |
| `domain` | string | 是 | 技能所属领域，如：音乐治疗、心理学、互联网运营、视频制作等 |
| `proficiency` | string | 是 | 熟练度级别，取值：入门、初级、中级、高级、专家 |
| `acquisition_date` | string | 是 | 首次获取该技能的日期，格式：YYYY-MM-DD |
| `source` | string | 是 | 知识来源，用于追溯和验证 |
| `version` | string | 否 | 版本号，默认：1.0 |
| `update_date` | string | 否 | 最后更新日期，格式：YYYY-MM-DD |
| `tags` | array | 否 | 标签列表，用于分类检索，建议 3-5 个 |
| `description` | string | 否 | 技能简介，不超过 100 字 |

#### 熟练度级别定义

| 级别 | 说明 |
| --- | --- |
| 入门 | 了解基本概念和术语，能在指导下完成简单任务 |
| 初级 | 掌握基础理论和方法，能独立完成常规任务 |
| 中级 | 精通核心技能，能解决复杂问题，有实践经验 |
| 高级 | 具备专业深度，能设计和优化方案，有创新能力 |
| 专家 | 领域权威，能引领方向，指导他人 |

### 2. 技能主体（必需）

技能主体为 Markdown 格式，内容结构应遵循以下推荐模板：

```markdown
# <技能名称>

## 技能概述

<简要介绍该技能的定义、核心价值和应用场景>

## 核心知识体系

### <知识模块 1>
<详细描述该知识模块的内容、要点和应用方法>

### <知识模块 2>
<详细描述该知识模块的内容、要点和应用方法>

### <知识模块 N>
<详细描述该知识模块的内容、要点和应用方法>

## 实践应用

### <应用场景 1>
<描述该技能在具体场景下的应用方法和案例>

### <应用场景 2>
<描述该技能在具体场景下的应用方法和案例>

## 注意事项

<列出使用该技能时需要注意的边界、限制和风险>

## 参考资料

<列出该技能的参考来源，如书籍、文章、网站等>
```

### 技能主体约束

- 内容应基于可靠来源，避免编造专业知识
- 使用清晰的标题层级（`##`、`###`）组织内容
- 关键术语可使用粗体强调
- 如有必要，可包含示例代码、图表或引用
- 文件总长度建议不超过 5000 字，超过时考虑拆分

## 保存时校验项

1. 元数据头部存在且格式正确
2. `name`、`domain`、`proficiency`、`acquisition_date`、`source` 字段不为空
3. `proficiency` 属于白名单（入门、初级、中级、高级、专家）
4. 日期格式为 `YYYY-MM-DD`
5. 文件名与 `name` 字段一致（不含 `.md` 后缀）
6. 技能主体至少包含 `## 技能概述` 和 `## 核心知识体系` 两个章节

## 版本管理

- 技能内容更新时，递增版本号
- 更新 `update_date` 字段为当前日期
- 重大变更时，可创建新版本文件（如 `音乐疗愈技能_v2.md`）
- 旧版本文件可保留作为历史参考

## 索引同步

技能文件创建或更新后，应同步更新 `EXPERTISE_INDEX.md`：

- 添加或更新技能条目
- 确保索引中的标签与技能文件一致
- 更新最后更新时间
