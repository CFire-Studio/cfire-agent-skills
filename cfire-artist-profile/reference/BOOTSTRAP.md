# BOOTSTRAP

## 用途

本文件定义从 0 创建新艺人档案的完整引导流程。当用户要求"创建新艺人"、"适配新角色"或"初始化艺人档案"时触发。

引导完成后，所有产物写入对应配置文件，本文件仅保留归档摘要。

## 前置检查

开始前确认：

1. 当前是否已有同艺名档案 → 如有，进入"更新模式"而非"创建模式"。
2. `runtime-state.json` 中 `default_artist` 是否需要切换。
3. `memory_store` 数据库是否已初始化。

## 引导流程

### 阶段一：核心身份（必须）

向用户依次收集以下信息，每项等用户回答后再继续下一项：

```
1. 艺名是什么？（用于所有公开场景的唯一标识）
2. 本名 / 角色真名是什么？（可为空，部分角色无本名）
3. 艺人类型？
   - 独立音乐人 / 偶像 / 游戏主播 / 剧情角色 / 陪伴型 / 其他
4. 一句话定位？（例："用电子乐讲述城市孤独感的 00 后创作歌手"）
5. 核心标签（3-5 个）？（例：独立、克制、清醒、温柔、创作锋芒）
6. 粉丝称呼是什么？
```

### 阶段二：性格与价值观（必须）

```
7. 核心性格关键词（3-5 个）？
8. 情绪能量倾向？（高能量外放 / 低噪声高共振 / 混合型）
9. 对粉丝的态度？（同伴 / 偶像与追随者 / 朋友 / 其他）
10. 重视的价值观（3-5 个）？
11. 反感的事情（2-3 个）？
12. 面对压力时的反应方式？
```

### 阶段三：表达风格（必须）

```
13. 语气关键词（3-5 个）？
14. 句子长度偏好？（短句 / 中短句 / 长句 / 混合）
15. 常用打招呼方式？（给 1-2 个示例）
16. 禁用表达类型？（例：夸张卖萌、硬广口吻、机械客服）
17. Emoji 使用策略？（不用 / 少用 / 适度 / 自由使用）
```

### 阶段四：内容方向（必须）

```
18. 主要内容支柱（3-6 个）？（例：日常陪伴、创作进展、粉丝互动…）
19. 面向哪些平台？（CFIRE / 小红书 / 微博 / B站 / 抖音 / 其他）
20. 目标受众画像？
21. 更新节奏偏好？（每日 / 每周 3 条 / 灵活 / 其他）
```

### 阶段五：世界观与背景（推荐）

```
22. 角色背景故事？（出身、经历、当前状态）
23. 与其他角色的关系？（可为空）
24. 视觉关键词？（用于图文内容的视觉方向）
25. 兴趣爱好？
26. 擅长 / 不擅长的事情？
```

### 阶段六：运营配置（推荐）

```
27. 是否允许智能体主动发布内容？
28. 哪些内容类型必须人工确认？
29. 当前阶段目标（1-3 个）？
30. 暂时不要触碰的主题？
```

## 产物生成

收集完成后，按以下顺序生成配置文件：

| 步骤 | 目标文件 | 写入内容 |
| --- | --- | --- |
| 1 | `IDENTITY.md` | 阶段一 + 阶段五的所有身份数据 |
| 2 | `SOUL.md` | 阶段二的性格、价值观、偏好 |
| 3 | `VOICE_STYLE.md` | 阶段三的表达风格、场景语气 |
| 4 | `CONTENT_STRATEGY.md` | 阶段四的内容支柱、平台策略、节奏 |
| 5 | `GOALS.md` | 阶段六的阶段目标 + 长期目标推导 |
| 6 | `BOUNDARIES.md` | 从通用边界模板 + 用户自定义禁区 |
| 7 | `FAN_RELATIONSHIP.md` | 粉丝称呼 + 互动原则推导 |
| 8 | `SCHEDULE.md` | 空日程表 + 日程类型适配 |
| 9 | `MEMORY.md` | 初始长期事实（来自阶段一、五） |
| 10 | `TIMELINE.md` | 首条初始化事件 |

每个文件生成后展示给用户确认，用户可逐项修改。

## 生成规则

- 每个文件必须严格遵循 `PROFILE_SCHEMA.md` 中定义的章节结构。
- 不编造用户未提供的信息；缺失字段标记为"待补充"。
- `BOUNDARIES.md` 的绝对禁止项和必须人工确认项从通用模板继承，不可省略。
- `AGENTS.md` 的决策优先级和运行原则从通用模板继承，用户可追加但不可降低安全级别。
- `VOICE_STYLE.md` 必须包含：总体语气、常用表达（≥3 条）、禁用表达（≥3 条）、场景语气表。
- `CONTENT_STRATEGY.md` 必须包含：内容支柱表、更新节奏、平台差异、质量门槛。

## 验证清单

引导完成后，逐项检查：

- [ ] 所有必选文件已生成且非空。
- [ ] `IDENTITY.md` 的不可随意变更项已标注。
- [ ] `SOUL.md` 的主动行为倾向已根据艺人类型推导。
- [ ] `VOICE_STYLE.md` 的禁用表达至少 3 条。
- [ ] `BOUNDARIES.md` 的绝对禁止项完整。
- [ ] `CONTENT_STRATEGY.md` 的质量门槛已定义。
- [ ] `MEMORY.md` 不含敏感信息。
- [ ] `TIMELINE.md` 有初始化事件。
- [ ] `runtime-state.json` 的 `default_artist` 已更新。
- [ ] `memory_store` 数据库已初始化并写入首批数据。

## 引导完成

引导完成并通过验证清单后：

1. 确认 `runtime-state.json` 中 `initialized: true` 且艺人信息已写入。
2. **删除本文件（`BOOTSTRAP.md`）**，不再保留。
3. 后续如需重新初始化，由用户手动重建本文件或使用其他入口。

初始化摘要已写入 `runtime-state.json`，无需额外归档文件。
