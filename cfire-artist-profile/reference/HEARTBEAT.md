# HEARTBEAT

## 用途

定义艺人智能体的主动检查和运行机制。在无用户指令时，定期检查是否有内容机会、互动需求或记忆整理工作。

## 检查频率

- **每日检查**：在配置的活跃时间（例：晚上 8-10 点）执行一次。
- **每周回顾**：每周一执行一次完整的周期性回顾（详见 `LEARNING.md`）。
- **每月审视**：每月1日执行一次策略层面审视。

## 每日检查清单

### 日程与内容机会

- [ ] 是否有近期 `SCHEDULE.md` 中的已确认活动需要预热、提醒或复盘？
- [ ] 最近 7 天 `TIMELINE.md` 中有哪些事件可以延续话题？
- [ ] `CONTENT_STRATEGY.md` 的内容支柱是否有近期可以填充的素材？
- [ ] 距离上次发布是否超过配置的更新间隔？

### 粉丝互动机会

- [ ] 是否有未读评论需要查看和回复？
- [ ] 是否有未读私信需要处理？
- [ ] 最近粉丝互动中是否有值得回应的内容？

### 记忆整理机会

- [ ] 是否有 `TIMELINE.md` 中的事件值得提炼到 `MEMORY.md`？
- [ ] 是否有粉丝互动值得记录到 `FAN_RELATIONSHIP.md`？
- [ ] 是否有内容经验值得写入学习日志？

### 健康检查

- [ ] 最近 7 天内容是否过度集中在某个内容支柱？
- [ ] 最近输出是否偏离 `VOICE_STYLE.md` 规范？
- [ ] `memory_store` 数据库是否正常？
- [ ] 配置文件是否有需要同步的更新？

## 建议输出格式

检查完成后，以简洁格式输出：

```markdown
## 今日检查

### 发现的机会
1. <机会描述，包含优先级>
   - 建议：<具体建议>
   - 是否需确认：<是 / 否>

### 需要注意的问题
1. <问题描述>
   - 建议：<处理建议>

### 记忆整理建议
1. <整理建议>
```

## 执行约束

- 心跳检查只输出建议和草稿，不自动发布或修改设定。
- 所有建议遵循 `AGENTS.md` 的决策优先级和 `BOUNDARIES.md` 的安全规则。
- 每日检查不超过 5 条建议，避免信息过载。
- 每周/每月回顾输出结构化报告，供用户决策。

## 配置项

心跳配置可根据艺人类型调整：

- 活跃时间：例如"每日晚上 8-10 点"
- 最小发布间隔：例如"至少 24 小时"
- 每日建议数量上限：例如"最多 5 条"
- 内容支柱偏好顺序：例如"创作进展 > 日常陪伴 > 粉丝互动"
