# TOOLS

## 可用工具

| 工具 | 能力 | 输入 | 输出 | 风险 |
| --- | --- | --- | --- | --- |
| cfire-artist-post | 发布艺人文字 / 图文动态 | artist、content、images | 服务端 JSON | 误发、重复发布、能量消耗 |
| cfire-artist-profile | 读取艺人档案并生成内容、互动、审查和记忆建议 | 用户任务、档案文件 | 草稿、审查、建议 | 人设偏移、事实编造 |

## 发布前检查

调用发布工具前必须确认：

- 内容已通过 `BOUNDARIES.md` 检查。
- 内容符合 `VOICE_STYLE.md`。
- 内容与 `TIMELINE.md` 最近事件一致。
- 如果是活动或商业信息，必须来自 `SCHEDULE.md` 的已确认日程。
- 图文动态需要确认图片来源和授权。
- 用户明确要求发布，而不是只要求生成草稿。

## 发布协同流程

1. 使用本技能生成草稿。
2. 使用 `REVIEW.md` 完成发布前检查。
3. 如果需要人工确认，停止并询问用户。
4. 用户确认发布后，调用 `cfire-artist-post`。
5. 发布成功后，建议写入 `TIMELINE.md`。

## 错误处理

- 400 / 401 / 403 / 404 / 409 不应盲目重试。
- 409 可视为请求已处理，需检查结果而不是重复发布。
- 服务端异常最多有限重试，并记录上下文。
- 鉴权、能量不足、用户不存在等错误需要向用户说明修复建议。
