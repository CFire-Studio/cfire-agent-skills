# REVIEW

## 用途

发布前审查机制与发布后复盘规范。确保内容安全、人设一致、质量达标，并从每次发布中积累经验。

## 发布前检查清单

每次发布前，逐项检查：

### 安全与合规

- [ ] 是否违反 `BOUNDARIES.md` 的绝对禁止项？
- [ ] 是否需要人工确认但未确认？
- [ ] 是否有编造的未确认事实、合作或活动？
- [ ] 是否有未授权使用的图片、音乐或内容？
- [ ] 是否涉及敏感个人、品牌或机构评价？
- [ ] 是否有隐私泄露风险？

### 人设一致性

- [ ] 是否符合 `IDENTITY.md` 的身份定位？
- [ ] 是否符合 `SOUL.md` 的性格与价值观？
- [ ] 语气和表达是否符合 `VOICE_STYLE.md`？
- [ ] 是否与 `MEMORY.md` 中的长期事实冲突？
- [ ] 是否与 `TIMELINE.md` 中最近事件连贯？

### 内容质量

- [ ] 是否符合 `CONTENT_STRATEGY.md` 的质量门槛？
- [ ] 内容支柱是否合理？
- [ ] 是否有低质量、凑数内容？
- [ ] 平台差异是否已考虑？

### 发布准备

- [ ] 如果是活动，是否在 `SCHEDULE.md` 中状态为"已确认"？
- [ ] 图文动态的图片来源和授权是否确认？
- [ ] 用户是否明确要求发布（而非仅生成草稿）？

### 发布决策

- **通过**：所有检查项通过 → 可以发布。
- **待确认**：存在需要人工确认的项 → 列出问题，等待用户确认。
- **不通过**：存在安全或人设一致性问题 → 列出风险，给出改写方案，停止发布。

## 发布后复盘

每次发布后，记录复盘：

```markdown
## 复盘 / YYYY-MM-DD

### 发布信息
- 内容类型：<类型>
- 平台：<平台>
- 发布时间：<时间>
- 初始状态：<草稿 / 待确认 / 已确认发布>

### 结果与反馈
- 客观结果：<播放/点赞/评论/互动数据>
- 主观反馈：<用户、粉丝或运营的评价>
- 问题或异常：<如有>

### 经验总结
- <3-5 条做得好的地方>
- <2-3 条可以改进的地方>

### 学习候选
- <学习建议，详见 LEARNING.md>
```

## 复盘归档

- 短期复盘（7天内）：保存在 `REVIEW.md` 中。
- 长期复盘（超过7天）：迁移到 `MEMORY.md` 作为内容经验。
- 严重问题复盘：必须在 `memory_store` 中保存为审查记录，便于后续检索和警示。

## 问题升级流程

如果发现以下情况，立即停止并请求人工确认：

1. 内容可能违反 `BOUNDARIES.md`。
2. 用户或粉丝反馈有强烈负面情绪。
3. 内容涉及争议性话题或敏感事件。
4. 人设一致性出现严重偏差。
